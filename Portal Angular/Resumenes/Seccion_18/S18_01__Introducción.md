# Lección 01: Introducción a la Sección 18

## ¿Qué se aprende en esta lección?

Esta lección es una introducción a la Sección 18 del curso. El instructor explica que a partir de aquí se construirá una aplicación Angular completa y real llamada **TesloShop**, cuyo propósito es consolidar y ampliar el conocimiento adquirido hasta ahora.

---

## Puntos clave explicados

- **Inicio del proyecto grande:** A partir de esta sección se trabajará en una aplicación real de Angular, de mediana a gran escala, para practicar con situaciones del mundo real.

- **Temas que se verán próximamente:** El instructor anticipa varios conceptos importantes que se aprenderán durante el desarrollo del proyecto:
  - Interceptores: bloques que permiten interceptar y modificar peticiones HTTP.
  - Librerías de terceros: cómo usar herramientas externas que no están escritas en Angular.
  - Estructura modular: organización del proyecto en módulos para escalabilidad.
  - Interfaces, paginaciones y trabajo con APIs.
  - Reutilización de componentes y servicios.
  - Carga de archivos.
  - Protección de rutas.
  - Fuentes personalizadas y diseño elegante usando recursos gratuitos (open source).
  - Uso básico de Docker para montar imágenes y contenedores.

- **Componentes standalone:** Se menciona que Angular ha evolucionado y ya no es necesario usar módulos para todo; ahora todo puede ser "standalone", lo que simplifica el desarrollo.

---

## Ejemplo sencillo

En esta lección no se muestra un ejemplo práctico explícito. Es una clase introductoria y motivacional.

---

## Funciones, palabras clave o elementos importantes

| Término | Descripción |
|--------|-------------|
| **Interceptores** | Mecanismo de Angular para interceptar peticiones antes de que lleguen al servidor o al cliente. |
| **Módulos (feature modules)** | Agrupaciones lógicas de componentes, servicios y otros elementos relacionados. |
| **Standalone components** | Componentes de Angular que no necesitan pertenecer a un módulo. Hace el código más simple. |
| **Paginación** | Técnica para mostrar información dividida en páginas en lugar de toda de golpe. |
| **Interfaces** | Estructuras de TypeScript que definen la forma que tendrá un objeto de datos. |
| **Docker** | Herramienta para crear entornos de software aislados llamados contenedores. |

---

## Resumen final en pocas palabras

Esta lección introduce la Sección 18, donde se construirá una tienda online completa con Angular para practicar conceptos avanzados reales. El objetivo es que el estudiante aprenda a pensar de forma modular y reutilizable, usando herramientas modernas del ecosistema frontend.
