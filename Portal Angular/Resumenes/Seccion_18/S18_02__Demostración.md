# Lección 02: Demostración del Proyecto TesloShop

## ¿Qué se aprende en esta lección?

El instructor muestra una demostración visual de lo que se va a construir durante la sección. Se presenta la aplicación terminada para que el estudiante tenga una idea clara del objetivo final antes de comenzar a desarrollarla.

---

## Puntos clave explicados

- **Objetivo de la sección:** Crear una página que consuma información desde una API (un backend ya construido) y la muestre de forma visualmente atractiva.

- **Backend proporcionado:** El instructor facilita un backend hecho en otro curso. No hay que crearlo desde cero; simplemente se usará para practicar Angular.

- **Swiper o slideshow:** La aplicación mostrará imágenes de productos en un carrusel o deslizador interactivo.

- **Diseño responsive:** La aplicación funcionará bien tanto en computadoras como en dispositivos móviles.

- **Paginación:** Los productos se mostrarán de nueve en nueve, con la posibilidad de navegar entre páginas.

- **Reutilización de pantallas:** Se usará la misma pantalla para mostrar productos filtrados por categoría (mujeres, niños, unisex, hombres).

- **Estructura robusta:** La arquitectura del proyecto estará pensada para aplicaciones medianas y grandes, usando **feature modules** (módulos por función).

- **Sin módulos de Angular tradicionales:** Todo se hará con **standalone components**, que es la forma moderna de trabajar en Angular.

- **Recursos adicionales:** Se enseñará a usar recursos estáticos, cambiar fuentes, aplicar temas y hacer animaciones como "fade in" y "fade out".

---

En esta lección no se muestra un ejemplo práctico explícito. Es una clase de demostración visual del producto final.

---

## Funciones, palabras clave o elementos importantes

| Término | Descripción |
|--------|-------------|
| **API** | Servicio externo del que se obtiene información, como los productos de la tienda. |
| **Swiper / Slideshow** | Componente visual que muestra imágenes deslizándose una tras otra. |
| **Paginación** | Dividir los resultados en páginas para no mostrar todo de una vez. |
| **Responsive** | Que se adapta correctamente a distintos tamaños de pantalla. |
| **Feature modules** | Carpetas o módulos que agrupan todo lo relacionado a una funcionalidad (ej: "productos"). |
| **Standalone components** | Componentes de Angular que funcionan de forma independiente, sin necesitar un módulo. |
| **Fade in / Fade out** | Animaciones de entrada y salida donde un elemento aparece o desaparece de forma gradual. |
