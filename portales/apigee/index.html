Práctico

Lo que aprenderás

Módulo 1 – Fundamentos de Integración y APIs
  • ¿Qué es una API y qué significa API?
  • Principios HTTP y REST
  • Formatos de datos: JSON, XML y YAML
  • Conceptos de API Management
  • Introducción a Google Apigee
  • Conceptos base: API Proxy, Shared Flows, Environment Configuration, Products y Apps
Módulo 2 – Políticas y configuración básica
  • AssignMessage
  • ExtractVariables
  • RaiseFault
  • VerifyAPIKey
  • OAuthV2
  • Quota
  • JavaScript
  • ServiceCallout
Módulo 3 – Políticas avanzadas y seguridad
  • FlowCallout
  • JSONToXML / XMLToJSON
  • LookupCache / PopulateCache
  • Operaciones con Key Value Maps (KVM)
  • CORS y configuración de seguridad para frontends
  • Basic Authentication
  • JSON Threat Protection
  • XML Threat Protection
Módulo 4 – Laboratorios prácticos
  • Hola Mundo – Proxy sin Target Server
  • Creación Producto, Developer y App
  • OAuthV2 – Seguridad JWT
  • WorkCafe API – Exposición servicio REST
  • Meli API – Consumo y exposición API pública
  • OMDb API – Integración API externa
  • Creación Target Server OMDb
  • Lemontech – Pass Through
  • Clave AFP – Transformación
  • Retail Connect – Transformación
  • OAuth Cache – Gestión y almacenamiento de tokens
  • Consulta en Línea (mTLS) – Certificados mutuos
🎯 Resultados Esperados
  • Comprender arquitectura y conceptos de API Management en Apigee.
  • Crear, documentar y exponer APIs seguras utilizando políticas estándar.
  • Aplicar transformaciones, autenticación y manejo de errores.
  • Implementar caching, KVM y protección contra amenazas.
  • Desarrollar APIs empresariales listas para despliegue y monitoreo.

Evaluación Teorica

Prueba Teorica

Evaluación de conocimientos conceptuales en Apigee Esta prueba mide la comprensión sólida de los fundamentos de Apigee, incluyendo arquitectura, diseño de APIs, políticas de control, seguridad, versionamiento y buenas prácticas en entornos regulados. Evalúa la capacidad del candidato para explicar conceptos clave como API Proxy, Target Endpoint, Shared Flows, mecanismos de autenticación, control de tráfico (Quota, Spike Arrest), validaciones previas al backend y estructura jerárquica de la plataforma. Su objetivo es validar criterio técnico, claridad conceptual y dominio de principios de API Management.

Descargar
Evaluación Técnica

Prueba Técnica Práctica

Esta prueba valida la capacidad del candidato para diseñar, configurar e implementar soluciones reales en Apigee bajo estándares de seguridad, control y transformación de datos. Incluye la creación de API Proxies, configuración de Target Servers, validación de contratos mediante OAS, control de cuotas, verificación de API Keys, transformación de datos (XML a JSON, Base64), orquestación con múltiples backends, uso de Shared Flows y manejo de cache con federación de tokens. El objetivo es evaluar habilidades prácticas, buenas prácticas de configuración, control de errores y capacidad de construir una capa de integración robusta y productiva.
