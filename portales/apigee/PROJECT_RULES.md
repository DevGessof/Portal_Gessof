## 3. Estructura del curso en Drive

Cada seccion del curso contiene archivos con el siguiente patron:

N.video_nombre.mp4
N.video_nombre.srt
N.X_material_descarga.html
N.X_imagen_referencia.jpg
N.entre otros


Donde:
- `N` corresponde al numero del video dentro de la seccion
- Todos los archivos que comienzan con el mismo numero pertenecen al **mismo video**
- El `.mp4` se usa para reproduccion
- El resto de archivos son **material de apoyo** y deben mostrarse como:
  - enlaces
  - recursos descargables
  - vistas embebidas si son HTML
  
- Archivo Drive.csv contiene todo la informacion de drive para poder acceder a ellos.
- Estrucutra del archivo:
   Numero de seccion
   Nombre de la seccion
   Nombre de archivo
   Id de google drive 
   Tipo de archivo
- Separado por punto y coma (;)

## 4. Estructura del proyecto web

- Existe un archivo `index.html` base que **debe respetarse visual y estructuralmente**.
- El proyecto debe ser **modular**, evitando:
  - un solo HTML gigante
  - logica mezclada con contenido
- Separacion obligatoria:
  - HTML (estructura)
  - CSS (estilos)
  - JavaScript (logica) separada por seccion
  - Datos / configuracion

## 5. Secciones del portal

El sitio tiene 4 secciones principales:

### 5.1 Inicio
Debe mostrar:
- Resumen general del curso
- Objetivos de aprendizaje
- El siguiente contenido textual debe incluirse y respetarse:

**Lo que aprenderas**
-Conocer el manejo y desarrollo de uno de los lenguaje de programación más usado en el mercado laboral
-Se realizará 100% práctico el aprendizaje de este curso.
-ejemplos claros al momento de nuestro proceso de desarrollo.
-Será nuestra formación base en el mundo del lenguaje de programación
---

### 5.2 Herramientas
- Debe listar todas las herramientas necesarias para el curso.
- Cada herramienta debe incluir:
  - Nombre
  - Descripcion breve
  - **Link oficial actualizado a la fecha**
- Es obligatorio buscar los links en internet (fuentes oficiales). 
- En la pagina index.html se hace referencia a las aplicacion base de otro curso, estas deben ser reemplazadas por las correctas segun este curso.
---

### 5.3 Curso (seccion principal)

- Cada seccion del curso debe mostrarse de forma independiente.
- Cada seccion debe tener:
	- Un carrusel de videos
	- Un carrusel por seccion (no uno global)
	- Sistema de Progreso: Persistencia del estado "Visto" de cada video utilizando `localStorage`, donde cada vez que se entre
	  se debe poder chequear el avance del usuario. En un futuro esta modalidad sera migrada a un documento en GoogleSheets.
	- Debe incluir checkboxes visuales en cada tarjeta y una barra de progreso global, la cual al chequear todos los videos debe marcar el 100%
	- Videos Favoritos/Importantes: Capacidad de marcar clases específicas como "Importantes" (estrella) y que esto persista.
	- Cada resumenes_seccion_X_java.txt donde X corresponde al numero de seccion, contiene todo el resumen de la seccion, es necesario agregar solamente
      el resumen asociado a cada video, de manera  didactica, visualmente atratcitvo para el usuario, separado en seccion, no solo texto plano en el resumen.
	
  


### 5.4 Glosario
Contendra un Glosario con las palabras, funcionalidades, y cosas mas importantes del curso.
- La informacion para llenar el glosario debe ser tomada estrictamente del archivo Resumen_Seccion/glosario_java_completo.txt
- Buscador de Glosario en Tiempo Real: El glosario debe tener un buscador instantáneo que filtre por término y descripción, 
  construido sobre la estrucutra de datos creada y buscando sobre los termino que se obtubieron del archivo glosario_apigee_completo.txt
  
  
###NOTA:
  Para garantizar que el portal funcione sin servidor (localmente), el contenido de los archivos de texto (resúmenes y glosario) 
  debe ser migrado a constantes de JavaScript (por ejemplo, en un archivo data.js) en lugar de cargarse dinámicamente mediante fetch. 
  No se deben hacer peticiones HTTP para leer los textos

## 6. Videos y resumenes

Para cada video:

- El video debe poder reproducirse desde Google Drive, con el link adjunto en documento Drive.csv
- Debe mostrarse un **resumen asociado al video**.
- El resumen debe:
	- Ser extraído de Resumen_Seccion/resumenes_seccion_X_java.txt
	- Cada seccion tiene su propio resumen (la X indica la seccion), el cual dentro tiene cada resumen por video.
	- Debe copiarse tal cual, ser distribuido de manera didactica, facil de entender y manejar.
---

## 7. Material de apoyo

- Todo material adicional del mismo numero del video debe mostrarse:
  - como enlaces
  - como recursos embebidos si corresponde
- No se debe ignorar ningun archivo de apoyo.

## 7.1 Manejo Inteligente de Material HTML

- **Excepción para Archivos HTML**: El sistema debe detectar automáticamente cuando un material de apoyo es un archivo `.html`.
- **Regla de Redirección**: Para estos archivos, **está prohibido enlazar a la descarga predeterminada de Google Drive**, 
	ya que esto obliga al usuario a descargar el archivo en lugar de verlo.
- **Acción Requerida**: El enlace debe comportarse como una **redirección directa a la página renderizada** (Vista Web). 
	El sistema debe permitir definir o mapear estos recursos para que se abran como páginas web funcionales en una nueva pestańa, 
	permitiendo al usuario interactuar con el contenido HTML en lugar de descargarlo como un archivo estático.
